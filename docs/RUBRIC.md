# novel-agent 审稿 Rubric

本文档定义 MVP 阶段章节审稿、通过线和返工策略。

## 评分维度

每项 0 到 10 分，最终总分换算为 100 分。

| 维度 | 字段 | 说明 |
| --- | --- | --- |
| 开头吸引力 | `opening_hook_score` | 开篇是否快速承接冲突、提出问题或制造期待 |
| 情节推进 | `pacing_score` | 本章是否有明确事件推进，而不是原地解释 |
| 爽点回报 | `payoff_score` | 是否给读者胜利、反击、揭秘、情绪释放或关系变化 |
| 人物表现 | `character_score` | 人物行动是否主动、可信、有差异 |
| 对话自然度 | `dialogue_score` | 对话是否带有目的、张力和角色口吻 |
| 设定一致性 | `continuity_score` | 是否违背已有人物、世界规则、事实和伏笔 |
| 章尾钩子 | `cliffhanger_score` | 结尾是否自然制造下一章期待 |
| 平台适配度 | `platform_fit_score` | 是否符合目标平台节奏和读者期待 |

## 默认通过线

章节通过必须同时满足：

```text
total_score >= 75
cliffhanger_score >= 7
continuity_score >= 8
pacing_score >= 7
```

任一硬性条件不满足，则 `passed = false`。

## 评分锚点

### 9 到 10 分

- 该维度表现强，能主动制造读者期待。
- 问题很少，即使修改也只是精修。

### 7 到 8 分

- 达到连载可发布标准。
- 存在轻微瑕疵，但不影响阅读动力。

### 5 到 6 分

- 勉强可读，但存在明显拖沓、空泛或弱冲突。
- 需要局部返工。

### 0 到 4 分

- 影响章节成立，读者可能弃读。
- 通常需要整章重写或重构核心冲突。

## 返工策略

### 局部返工

适用条件：

- 总分达标，但某一两个维度偏弱。
- 连续性无硬伤。
- 章节主事件可保留。

处理方式：

- 强化开头冲突。
- 压缩解释性段落。
- 增加对手施压或代价。
- 重写章尾钩子。
- 调整对白，让人物目标更明确。

### 整章重写

适用条件：

- `total_score < 65`
- `continuity_score < 6`
- 章节没有实质推进。
- 主角行为明显违背人物卡。
- 章尾没有下一章期待，且无法靠局部修改解决。

处理方式：

- 保留必要事实和大纲目标。
- 重构本章冲突链。
- 重新设计事件顺序和情绪回报。
- 重写正文后再次执行连续性检查和审稿。

## 平台差异

### 起点向

更看重：

- 设定可信度
- 升级体系
- 长线伏笔
- 阶段目标
- 主角策略和成长

常见扣分：

- 体系规则随意变化
- 主角升级没有代价
- 长线目标模糊
- 反派只会送经验

### 番茄向

更看重：

- 开篇速度
- 情绪反馈
- 关系张力
- 短周期爽点
- 易读性

常见扣分：

- 解释过长
- 前三章冲突不足
- 爽点兑现慢
- 章尾期待弱

## 审稿输出原则

- 先判断是否能发布，再谈文学性。
- 问题必须定位到具体段落、事件或维度。
- 建议必须可执行，避免只写“加强节奏”“人物更立体”。
- 低分必须触发明确返工指令。
