# novel-agent 审稿 Rubric

本文档定义 MVP 阶段章节审稿、通过线和返工策略。

## 评分维度

每项 0 到 10 分，最终总分换算为 100 分。

| 维度 | 字段 | 说明 |
| --- | --- | --- |
| 开头吸引力 | `opening_hook_score` | 开篇是否快速承接冲突、提出问题或制造期待 |
| 情节推进 | `pacing_score` | 本章是否有明确事件推进，而不是原地解释 |
| 爽点回报 | `payoff_score` | 是否给读者胜利、反击、揭秘、情绪释放或关系变化 |
| 人物表现 | `character_score` | 人物行动是否主动、可信、有差异 |
| 对话自然度 | `dialogue_score` | 对话是否带有目的、张力和角色口吻 |
| 设定一致性 | `continuity_score` | 是否违背已有人物、世界规则、事实和伏笔 |
| 章尾钩子 | `cliffhanger_score` | 结尾是否自然制造下一章期待 |
| 平台适配度 | `platform_fit_score` | 是否符合目标平台节奏和读者期待 |

## 默认通过线

章节通过必须同时满足：

```text
total_score >= 75
cliffhanger_score >= 7
continuity_score >= 8
pacing_score >= 7
```

任一硬性条件不满足，则 `passed = false`。

## v0.3 一致性硬门

v0.3 Web 工作台会把 ReviewReport、Continuity、facts 和 AgentRun 同屏展示，因此以下跨产物不一致会比纯文本问题更显眼，必须作为硬性质量门处理：

- 主角姓名在 NovelBible、人物卡、大纲、正文、事实表中不一致，且正文没有解释为别名、谎言或误报。
- 关键金额不一致，例如罚款、欠薪、押金、订单金额、合同金额前后出现两个口径。
- 合作、敌对、合同、订单状态不一致，例如摘要说“已经谈妥”，正文仍是“疑似谈妥”或“尚未接触”。
- 核心人物关系状态不一致，例如前文是敌对，当前无解释地变成合作。

扣分规则：

- 出现上述任一硬伤，`continuity_score` 最高不得超过 6。
- 如果硬伤影响本章主冲突、章尾钩子或后续写作上下文，`passed = false`。
- `issues[]` 必须标为 `dimension = continuity`，并在 `suggestions[]` 中写清楚应统一成哪个姓名、金额或状态。
- `rewrite_instruction.change[]` 必须包含对应修复动作，避免 Web 端只看到“连续性有问题”却不知道怎么改。

## 评分锚点

### 9 到 10 分

- 该维度表现强，能主动制造读者期待。
- 问题很少，即使修改也只是精修。

### 7 到 8 分

- 达到连载可发布标准。
- 存在轻微瑕疵，但不影响阅读动力。

### 5 到 6 分

- 勉强可读，但存在明显拖沓、空泛或弱冲突。
- 需要局部返工。

### 0 到 4 分

- 影响章节成立，读者可能弃读。
- 通常需要整章重写或重构核心冲突。

## 返工策略

`rewrite_instruction.rewrite_type` 取值：

| 值 | 中文 | 适用条件 |
| --- | --- | --- |
| `none` | 无需返工 | 章节已过线，只需要人工轻修 |
| `partial` | 局部返工 | 主事件可保留，但存在局部节奏、爽点、对白或平台适配问题 |
| `full` | 整章重写 | 章节结构不成立、连续性硬伤或总分明显低于发布线 |
| `opening` | 开头重写 | 开篇目标不清、冲突进入太慢、前 800 字抓力不足 |
| `ending` | 结尾重写 | 章尾没有自然钩子，下一章期待弱 |
| `style` | 语言润色 | 事实和结构成立，但表达、对白、网文语感偏弱 |

### 局部返工

适用条件：

- 总分达标，但某一两个维度偏弱。
- 连续性无硬伤。
- 章节主事件可保留。

处理方式：

- 强化开头冲突。
- 压缩解释性段落。
- 增加对手施压或代价。
- 重写章尾钩子。
- 调整对白，让人物目标更明确。

### 整章重写

适用条件：

- `total_score < 65`
- `continuity_score < 6`
- 章节没有实质推进。
- 主角行为明显违背人物卡。
- 章尾没有下一章期待，且无法靠局部修改解决。

处理方式：

- 保留必要事实和大纲目标。
- 重构本章冲突链。
- 重新设计事件顺序和情绪回报。
- 重写正文后再次执行连续性检查和审稿。

## 平台差异

### 起点向

更看重：

- 设定可信度
- 升级体系
- 长线伏笔
- 阶段目标
- 主角策略和成长

常见扣分：

- 体系规则随意变化
- 主角升级没有代价
- 长线目标模糊
- 反派只会送经验

评分偏置：

- `continuity_score` 权重最高。设定自洽、能力边界清晰、伏笔可追踪时可上调 1 到 2 分。
- `platform_fit_score` 应重点看升级体系、长期期待和阶段目标。
- 如果体系规则自相矛盾，`continuity_score` 最高不得超过 6，即使本章爽点成立。
- 如果前十章仍没有长期目标或升级路径，`platform_fit_score` 最高不得超过 6。

### 番茄向

更看重：

- 开篇速度
- 情绪反馈
- 关系张力
- 短周期爽点
- 易读性

常见扣分：

- 解释过长
- 前三章冲突不足
- 爽点兑现慢
- 章尾期待弱
- 商业计划、未来趋势或设定说明多于场景行动
- 谈判、创业、合作推进过顺，没有对方阻力、失败代价或资源消耗

评分偏置：

- `opening_hook_score`、`pacing_score`、`payoff_score` 权重最高。开篇快、冲突直、回报密时可上调 1 到 2 分。
- `platform_fit_score` 应重点看易读性、短周期情绪反馈和章尾期待。
- 如果第 1 章前 800 字仍没有核心冲突，`opening_hook_score` 最高不得超过 6。
- 如果连续两章没有明确情绪回报或爽点，`payoff_score` 最高不得超过 6。
- 如果商业或谈判主线没有可见阻力和代价，`pacing_score` 与 `payoff_score` 最高不得超过 6。
- 如果章尾只表达长期方向而没有下一章具体压力，`cliffhanger_score` 最高不得超过 6。

### 通用向

更看重：

- 叙事清晰
- 人物目标明确
- 每章有推进
- 设定不过载
- 章尾自然制造期待

评分偏置：

- 不做极端加权。
- 任一核心维度低于 6 时，`platform_fit_score` 最高不得超过 7。

## 审稿输出原则

- 先判断是否能发布，再谈文学性。
- 问题必须定位到具体段落、事件或维度。
- 建议必须可执行，避免只写“加强节奏”“人物更立体”。
- 低分必须触发明确返工指令。
